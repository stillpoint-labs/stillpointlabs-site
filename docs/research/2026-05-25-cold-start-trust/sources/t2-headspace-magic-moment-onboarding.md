## Key Findings

1. **Deliver competence in the loading screen.** Headspace's loading state IS a breathing exercise. Before the user is asked a single question, they experience a value-delivering moment. This is the most-cited trust pattern in the corpus.
2. **The pre-post comparison is the Magic Moment.** Andy (the voice) asks the user to notice their internal state pre- and post-meditation. The user notices their own change; the product gets credit for delivering that change. Critically: the product doesn't TELL the user they should feel better; it ASKS them to check.
3. **"What brings you to Headspace?" — one question, picked from a short list.** Not a Big Five survey; a list of 4-5 goal categories (sleep, anxiety, focus, general). The answer is used as a routing signal, not as a stored personality label.
4. **The Basics course is choice-architectured.** 10 days, 3-5-10 minute options. The user picks the length and commits to a short sequence. Day 1 is in-session; day 2-10 build the habit. The "trust the system enough to come back tomorrow" hurdle is small.
5. **Doubling course starts came from a single behavioral-science change** (per the Irrational Labs case). Not from adding features. From reframing the first decision.
